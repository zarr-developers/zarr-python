`FsspecStore.close()` no longer closes the underlying fsspec filesystem or its
network session. fsspec caches and shares filesystem instances across callers,
so a store cannot know whether it is the only user, and closing a shared
session broke sibling stores built from the same host — and left the closed
filesystem in fsspec's cache for later callers. The filesystem's lifecycle now
stays with whoever created it; use fsspec's own tools (for example
`AbstractFileSystem.clear_instance_cache()`) to release it.

This removes the store-owns-the-filesystem behavior added in gh-4003. Code that
relied on `store.close()` releasing the filesystem's session must now close the
filesystem itself.
