Replaced hatch with uv for development environments and CI task management. Contributors now create the development environment with `uv sync --locked` and run tests with `uv run pytest` (or the shared runners in `ci/`, e.g. `bash ci/run-coverage.sh`); see the contributing guide for details. The `tool.hatch.envs.*` environments and their run scripts were removed from `pyproject.toml`, and a committed `uv.lock` now pins the full transitive dependency closure for reproducible, cross-platform CI. hatchling remains the build backend, so package builds and version derivation are unchanged.
