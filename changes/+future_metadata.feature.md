`zarr` now depends on the `zarr-metadata` package, and array and group classes gain a `_future_metadata` attribute exposing the node's metadata as a `zarr_metadata` document model (`ZarrV2ArrayMetadata` / `ZarrV3ArrayMetadata` for arrays, `ZarrV2GroupMetadata` / `ZarrV3GroupMetadata` for groups). These models are the planned future type of the public `metadata` attribute: canonical, lossless representations of the stored metadata documents, with extension points held uninterpreted. Reading the synchronous `Array.metadata` / `Group.metadata` properties now emits a `ZarrPendingDeprecationWarning` (ignored by default) announcing the pending type change.
